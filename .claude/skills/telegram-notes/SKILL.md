# telegram-notes

Скилл забирает только новые сообщения бота из Telegram (повторно обработанные
ранее апдейты не трогает — см. инкрементальную обработку по offset ниже).
Текстовые сообщения берутся как есть, голосовые расшифровываются локально
через Whisper. Весь текст делится на идеи и задачи и дописывается в конец
двух markdown-файлов.

## Когда использовать
Когда пользователь вызывает `/telegram-notes` или просит забрать заметки из Telegram.

## Предусловия
- В `.env` в корне проекта есть переменная `TELEGRAM_BOT_TOKEN`.
- Пользователь уже отправил боту сообщения (Telegram хранит апдейты ~24 часа).
- Для расшифровки голосовых нужен локальный Whisper (Python-пакет `openai-whisper`)
  и `ffmpeg`. Если их нет — установи (см. шаг 0). Облачные API распознавания
  речи использовать нельзя — только локальная модель.
- Файл `telegram-notes/.last_update_id` хранит `update_id` последнего
  обработанного апдейта (инкрементальная обработка, см. шаги 2 и 6). Он лежит
  внутри `telegram-notes/`, которая уже целиком в `.gitignore` (правило
  `/telegram-notes/`) — в git попадать не должен.

## Что нужно сделать

0. Проверь зависимости для расшифровки голосовых (делать это нужно один раз,
   до первого голосового сообщения в текущем запуске; если в апдейтах нет
   голосовых — можно пропустить этот шаг целиком):
   - Проверь версию системного Python. `openai-whisper` тянет `torch`/`numba`,
     у которых может не быть готовых сборок под самые новые версии Python
     (например, на момент написания — под 3.14 их нет, собирать из исходников
     не получится). Если системный Python новее 3.12 или установка
     `pip install openai-whisper` падает с ошибками сборки зависимостей —
     создай отдельное виртуальное окружение на Python 3.10–3.12
     (`py -3.11 -m venv <путь>`, например `.claude/skills/telegram-notes/.venv`)
     и ставь/запускай Whisper только в нём. Эту папку с venv не коммитить —
     добавь её в `.gitignore`.
   - Проверь, установлен ли `openai-whisper` (например, `python -c "import whisper"`
     или `pip show openai-whisper`). Если нет — установи: `pip install openai-whisper`.
   - Проверь наличие `ffmpeg` в PATH (`ffmpeg -version`). Если его нет:
     - Windows: попробуй `winget install ffmpeg` или `choco install ffmpeg`;
       если ни один менеджер не доступен — сообщи пользователю команду
       и остановись, не пытаясь продолжить расшифровку.
     - macOS: `brew install ffmpeg`.
     - Linux (Debian/Ubuntu): `sudo apt install ffmpeg`.
   - Модель по умолчанию — `base` (баланс скорости/качества). Она скачается
     автоматически и закешируется на диск при первом запуске Whisper —
     первый прогон может быть заметно дольше последующих.

1. Прочитай токен бота из `.env` (переменная `TELEGRAM_BOT_TOKEN`).
   Не выводи токен в чат.

2. Прочитай сохранённый offset перед запросом:
   - Проверь, существует ли файл `telegram-notes/.last_update_id`.
   - Если существует — прочитай из него число (`last_id`, это `update_id`
     последнего обработанного апдейта из предыдущего запуска).
   - Если файла нет (первый запуск) — `last_id` отсутствует, обрабатывай
     все апдейты, которые придут.

3. Получи сообщения через Telegram Bot API методом getUpdates:
   - Если `last_id` известен — вызови с параметром offset, чтобы Telegram
     сам отдал только новые апдейты:
     `https://api.telegram.org/bot<TOKEN>/getUpdates?offset=<last_id + 1>`
   - Если `last_id` не известен (первый запуск) — вызови без offset:
     `https://api.telegram.org/bot<TOKEN>/getUpdates`
   - Не подтверждай offset через отдельный вызов — обновление произойдёт
     через сохранение `.last_update_id` в шаге 6, а не через сам Telegram API.
   - Дополнительная страховка в коде: даже после ответа с offset, отфильтруй
     полученные апдейты и пропусти те, у которых `update_id <= last_id`
     (на случай, если offset почему-то не сработал на стороне Telegram).

4. Для каждого оставшегося после фильтра апдейта определи источник текста:
   - Если есть `message.text` — возьми текст напрямую (как раньше).
   - Если есть `message.voice` — НЕ пропускай, а расшифруй локально:
     a. возьми `file_id` из `message.voice.file_id`;
     b. вызови `getFile`: `https://api.telegram.org/bot<TOKEN>/getFile?file_id=<file_id>`,
        получи `file_path` из ответа;
     c. скачай файл по адресу
        `https://api.telegram.org/file/bot<TOKEN>/<file_path>`
        (формат `.oga`, кодек OPUS) во временную папку (например, системный
        temp), а не в репозиторий — эти файлы не должны попасть в git;
     d. прогони файл через локальный Whisper (модель `base`) и получи текст
        расшифровки;
     e. удали временный аудиофайл после расшифровки;
     f. дальше обращайся с расшифрованным текстом так же, как с обычным
        `message.text`.
   - Прочие нетекстовые и не-голосовые сообщения пропусти.

5. Раздели весь полученный текст (и набранный, и расшифрованный) на две группы:
   - **Задачи** — если в тексте есть слова-маркеры действия:
     задача, сделать, надо, нужно, настроить, реализовать, сверстать,
     собрать, подготовить, спроектировать, протестировать, срок, до конца, todo.
   - **Идеи** — всё остальное текстовое.
   - Если в сообщении есть и то и другое — считай задачей.

6. Сохрани результат — ДОПИШИ новые записи в конец файлов, не перезаписывай
   и не очищай существующее содержимое:
   - идеи → `telegram-notes/ideas.md`
   - задачи → `telegram-notes/tasks.md`
   Каждый пункт — отдельным маркером списка (`- текст сообщения`),
   добавленным новой строкой в конец файла. Папку `telegram-notes/`
   создай, если её нет; файлы создай, если их нет, но если они уже
   существуют — открывай в режиме дозаписи (append), а не перезаписи.
   Если за этот запуск не пришло ни одного нового апдейта — оба файла
   не трогай вообще (не открывай на запись, не создавай пустых версий).

7. Обнови сохранённый offset:
   - Если в этом запуске был обработан хотя бы один новый апдейт — найди
     максимальный `update_id` среди них и запиши его (одним числом,
     перезаписав предыдущее значение) в `telegram-notes/.last_update_id`.
   - Если новых апдейтов не было — файл `.last_update_id` не трогай,
     оставь прежнее значение.

8. Покажи пользователю сводку:
   - какой `last_id` (offset) был прочитан перед запросом (или "отсутствовал
     — первый запуск");
   - сколько новых апдейтов обработано (текстовых / голосовых расшифровано);
   - сколько идей и задач добавлено за этот запуск;
   - какой новый `last_update_id` сохранён (или что он не менялся, если
     новых апдейтов не было).
